# Логика управления инвертором

## Обзор

Этот документ описывает логику управления, используемую для расчёта уставки мощности, отправляемой в систему Victron. Цель — минимизировать импорт/экспорт из сети, соблюдая различные режимы работы.

## Соглашение об уставке

Для Victron в режиме **External Control**:
- **Положительная уставка** = Потребление из сети (зарядка батареи)
- **Отрицательная уставка** = Отдача в дом (разрядка батареи)

Примеры:
- `setpoint = -1000` → Инвертор отдаёт 1000W в дом
- `setpoint = +500` → Инвертор потребляет 500W из сети для зарядки

## Цикл управления

Основной цикл выполняется каждые ~0.33 секунды и включает следующие шаги:

### Шаг 1: Сбор входных данных

| Переменная | Источник | Описание |
|------------|----------|----------|
| `g1`, `g2` | D-Bus | Мощность сети по фазам (Вт) |
| `gt` | D-Bus | Общая мощность сети (Вт), положительное = импорт |
| `t1`, `t2`, `tt` | D-Bus | Потребление по фазам и общее (Вт) |
| `inv_power` | D-Bus | Текущая выходная мощность инвертора (Вт) |
| `mppt_total` | D-Bus | Солнце от MPPT контроллеров (Вт) |
| `tasmota_total` | D-Bus | Солнце от Tasmota микроинверторов (Вт) |
| `ev_power` | Home Assistant | Потребление зарядки EV (Вт) |

### Шаг 2: Получение переключателей управления

Все переключатели читаются из Home Assistant `input_boolean`:

| Переключатель | Назначение |
|---------------|------------|
| `only_charging` | Не разряжать батарею, использовать только солнце |
| `do_not_supply_charger` | Не питать EV от батареи |
| `no_feed` | Соответствовать выходу Tasmota для предотвращения экспорта |
| `house_support` | Tasmota минус 300W для частичного самопотребления |
| `charge_battery` | Принудительная зарядка батареи |

### Шаг 3: Расчёт эффективной сети

Когда `do_not_supply_charger` включен:
```
effective_gt = gt - ev_power
```

Это делает алгоритм "слепым" к потреблению EV, предотвращая разрядку батареи для питания зарядки автомобиля.

### Шаг 4: Базовый расчёт

Цель: Мощность сети ≈ 0 (grid-zero)

```
vanew = inv_power - effective_gt
```

Логика:
- Если импортируем 500W из сети → увеличить выход на 500W
- Если экспортируем 200W в сеть → уменьшить выход на 200W

Зона стабильности: Если `-30 < effective_gt < 50`, сохраняем предыдущую уставку для предотвращения осцилляций.

### Шаг 5: Применение режимов работы

Режимы применяются в порядке приоритета (от низшего к высшему):

#### 1. ONLY_CHARGING (Низший приоритет)
**Цель:** Не разряжать батарею - отдавать только то, что производят MPPT

```
output = mppt_total - SOLAR_OUTPUT_OFFSET
vanew = -max(0, output)
```

Флаг: `[OC:XXX-60]`

#### 2. DO_NOT_SUPPLY_CHARGER
**Цель:** Не позволять батарее питать зарядку EV

```
max_output = max(0, mppt_total - SOLAR_OUTPUT_OFFSET)
if vanew < -max_output:
    vanew = -max_output
```

Флаг: `[NoEV]`

#### 3. NO_FEED
**Цель:** Соответствовать выходу Tasmota для предотвращения экспорта в сеть

```
vanew = tasmota_total
```

Положительная уставка потребляет из сети то, что Tasmota экспортирует.

Флаг: `[NF]`

#### 4. HOUSE_SUPPORT
**Цель:** Частичное самопотребление с Tasmota

```
vanew = tasmota_total - 300
```

Флаг: `[HS]`

#### 5. CHARGE_BATTERY (Высший приоритет)
**Цель:** Принудительная максимальная зарядка батареи

```
vanew = 2200
```

Флаг: `[CHG]`

### Шаг 6: Применение лимитов безопасности

```
vanew = max(POWER_LIMIT_MIN, min(POWER_LIMIT_MAX, vanew))
```

Значения по умолчанию:
- `POWER_LIMIT_MIN = -2300` (максимальная отдача)
- `POWER_LIMIT_MAX = +2250` (максимальная зарядка)

## Параметры конфигурации

| Параметр | Значение | Описание |
|----------|----------|----------|
| `SOLAR_OUTPUT_OFFSET` | 60W | Уменьшить выход на эту величину для избежания экспорта |
| `LOOP_INTERVAL` | 0.33с | Интервал цикла управления |
| `POWER_LIMIT_MIN` | -2300W | Максимальная отдача (разрядка) |
| `POWER_LIMIT_MAX` | +2250W | Максимальная зарядка |

## Флаги в консольном выводе

| Флаг | Значение |
|------|----------|
| `[~]` | Сеть близка к нулю, держим стабильно |
| `[EV:XXX]` | Мощность EV вычтена из расчёта сети |
| `[OC:XXX-60]` | Режим только зарядка, MPPT минус смещение |
| `[NoEV]` | Применено ограничение исключения EV |
| `[NF]` | Режим без отдачи активен |
| `[HS]` | Режим поддержки дома активен |
| `[CHG]` | Режим зарядки батареи активен |

## Обработка ошибок

- Отключение Home Assistant: Используются последние известные значения (кэш)
- Ошибки D-Bus: Автоматическое пересканирование сервисов после серии неудач
- Watchdog: Перезапуск сервиса если веб-сервер перестаёт отвечать
